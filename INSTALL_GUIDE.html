üì± Detailed Installation Guide

Option A: Using GitHub Pages (Recommended - Free)

  1. Create a free GitHub account at github.com
  2. Create a new repository (name it "tree-tracker")
  3. Upload all the files:
    • index.html
    • manifest.json
    • sw.js
    • icon-192.png
    • icon-512.png
  4. Go to Settings ‚Üí Pages
  5. Select "main" branch and save
  6. Your app will be available at: https://YOUR_USERNAME.github.io/tree-tracker
  7. Open this URL on your Android phone
  8. Install the app when prompted!

Option B: Using Netlify (Drag & Drop)

  1. Go to netlify.com
  2. Create a free account
  3. Drag and drop all 5 files into Netlify
  4. Get your app URL (like: your-app.netlify.app)
  5. Open on Android and install!

Option C: Local Network (For Testing)

  1. Put all files in a folder on your computer
  2. Use Python to start a server: 
    python3 -m http.server 8000
  3. Find your computer's IP address
  4. On Android, open: http://YOUR_IP:8000
  5. Note: This only works on the same WiFi network

üîß Troubleshooting

Problem: Install button doesn't appear
Solution: Make sure you're using Chrome browser and the site is HTTPS. Try using Chrome menu ‚Üí "Add to Home screen" instead.
Problem: Data disappears
Solution: Don't clear browser data/cache. Use the "Export Data" button regularly to backup. Make sure you're opening the same URL each time.
Problem: Photos won't upload
Solution: Make sure Chrome has camera/storage permissions. Photos are stored as base64 data and have a size limit (~5MB per photo).
Problem: App won't work offline
Solution: Open the app while online first to cache all resources. Service worker needs HTTPS to work properly.

üí° Tips & Best Practices

  • Regular Backups: Use "Export Data" weekly to backup your database
  • Photo Size: Compress large photos before uploading for better performance
  • Naming Convention: Use consistent plot names and tree numbers
  • Condition Notes: Add notes whenever you inspect trees - dates are automatic!
  • Bulk Import: Use CSV for initial setup, then add photos individually
  • Multiple Devices: Install on multiple devices, but remember data is per-device (export/import to sync)
